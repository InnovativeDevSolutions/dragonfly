# Installation

This guide will walk you through installing ArmaDragonflyClient and setting up DragonflyDB for your Arma 3 server or mission.

## Prerequisites

Before you begin, ensure you have:

- **Arma 3** (client or dedicated server)
- **DragonflyDB server** (see [DragonflyDB Setup](#dragonflydb-setup) below)
- Basic familiarity with Arma 3 modding

## Part 1: DragonflyDB Setup

ArmaDragonflyClient requires a DragonflyDB server to store data. DragonflyDB is available for Linux, macOS, and Windows.

### Option A: Docker (Recommended)

The easiest way to run DragonflyDB is using Docker:

```bash
docker run -d --name dragonfly -p 6379:6379 docker.dragonflydb.io/dragonflydb/dragonfly
```

### Option B: Native Installation

**Linux:**
```bash
# Ubuntu/Debian
curl -fsSL https://dragonflydb.io/install.sh | bash

# Start DragonflyDB
dragonfly --logtostderr
```

**Windows:**
Use Docker Desktop or WSL2 with the Linux instructions above.

**macOS:**
```bash
brew install dragonfly
dragonfly --logtostderr
```

### Option C: Cloud Hosted

For production servers, consider using [Dragonfly Cloud](https://www.dragonflydb.io/cloud) for managed hosting with automatic backups and scaling.

::callout{icon="i-lucide-info" color="secondary"}
**Default Port**: DragonflyDB runs on port 6379 by default (same as Redis). Make sure this port is accessible from your Arma 3 server.
::

Download ArmaDragonflyClient from one of these sources:

### Step 3: Create Configuration File

Create a `config.xml` file in the `@dragonfly` directory:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<config>
    <host>127.0.0.1</host>
    <port>6379</port>
    <password></password>
    <contextLog>false</contextLog>
    <debug>false</debug>
</config>
```

### Configuration Options

| Option | Type | Description | Default |
|--------|------|-------------|---------|
| `host` | String | DragonflyDB server IP address | 127.0.0.1 |
| `port` | Integer | DragonflyDB server port | 6379 |
| `password` | String | DragonflyDB authentication password (leave empty if none) | xyz123 |
| `contextLog` | Boolean | Log call context information (Steam ID, file source, etc.) | false |
| `debug` | Boolean | Enable detailed debug logging | false |

::callout{icon="i-lucide-alert-triangle" color="warning"}
**Security Note**: If your DragonflyDB server is exposed to the internet, always set a strong password and use `requirepass` in DragonflyDB configuration.
::

### Step 4: Enable the Mod

#### For Server:

Add the mod to your server startup parameters:

```bash
-mod=@dragonfly
```

**Example startup line:**
```bash
arma3server_x64.exe -mod=@dragonfly -config=server.cfg -port=2302
```

#### For Client:

1. Launch Arma 3
2. Go to **Launcher** ‚Üí **Mods**
3. Enable **@dragonfly**
4. Click **Play**

## Step 5: Verify Installation

Once Arma 3 launches, check your RPT log file to verify ArmaDragonflyClient loaded correctly:

**Location of RPT files:**
- **Client**: `C:\Users\[YourUsername]\AppData\Local\Arma 3\`
- **Server**: Usually in the same directory as the server executable

Look for these lines:

```
ArmaDragonflyClient 1.0.0
Config file found! Context Mode: false! Debug Mode: false! Host Address: 127.0.0.1! Host Port: 6379!
```

::callout{icon="i-lucide-check-circle" color="primary"}
**Success!** If you see these messages, ArmaDragonflyClient is installed and ready to use.
::

### Test the Connection

In your mission, run:

```sqf
[] call dragonfly_db_fnc_test;
```

You should see a confirmation message indicating the connection is working.

## Troubleshooting

### Mod Not Loading

::u-accordion
---
items:
  - label: "Extension not found"
    content: "Make sure the @dragonfly folder is in your Arma 3 root directory, not in the addons folder. The ArmaDragonflyClient_x64.dll (or .so) should be in the @dragonfly directory."
    defaultOpen: false
  - label: "Wrong directory structure"

    defaultOpen: false

### Connection Issues

::u-accordion
---
items:
  - label: "Cannot connect to DragonflyDB"
    content: "Verify DragonflyDB is running: `docker ps` or `ps aux | grep dragonfly`. Check that the host and port in config.xml match your DragonflyDB server."
    defaultOpen: false
  - label: "Authentication failed"
    content: "If DragonflyDB requires a password, make sure it matches the password in config.xml. Leave the password field empty if DragonflyDB doesn't require authentication."
    defaultOpen: false
  - label: "Connection timeout"
    content: "Check firewall settings. Port 6379 must be accessible from your Arma 3 server. Test connectivity: `telnet [host] 6379`"
    defaultOpen: false
---
::

### Config File Issues

If you see "Config file not found! Default settings will be used" in the logs:

1. Verify `config.xml` exists in `@dragonfly\config.xml`
2. Check the file is not `config.xml.txt` (Windows may hide extensions)
3. Ensure the XML structure matches the example above
4. Verify file permissions allow reading

## Performance Tuning

For high-traffic servers, consider these DragonflyDB optimizations:

```bash
# Increase memory limit (default is unlimited)
dragonfly --maxmemory=2gb

# Enable snapshotting for persistence
dragonfly --save_schedule "*:*" --dir=/var/lib/dragonfly

# Tune for specific workload
dragonfly --cache_mode=true  # For read-heavy workloads
```
